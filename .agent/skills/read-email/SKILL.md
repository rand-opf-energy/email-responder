# read-email Custom Skill

This skill allows Antigravity to programmatically search, fetch, and inspect emails inside your Gmail account using your local **Application Default Credentials (ADC)**. It is primarily used to validate the behaviors and responses generated by the `opf/email-responder` bot.

---

## 🚀 How to Run the Skill

To retrieve recent emails, run the following Node.js command in your terminal from the root of the `email-responder` repository:

```bash
node .agent/skills/read-email/scripts/read.js [options]
```

### Options:
*   `--query "<search-query>"`: Customize the Gmail search filter. Defaults to `"is:unread"`.
*   `--limit <number>`: Limit the maximum number of threads retrieved. Defaults to `5`.

---

## 💡 Example Queries & Common Tasks

### 1. View Unread Support Emails
Search for unread emails sent to the help address to see what the bot is about to process:
```bash
node .agent/skills/read-email/scripts/read.js --query "is:unread to:help@opf.energy" --limit 5
```

### 2. Verify Bot Responses
Check the recent replies sent by the bot (or `opti@opf.energy`) to confirm they match guidelines:
```bash
node .agent/skills/read-email/scripts/read.js --query "from:opti@opf.energy" --limit 3
```

### 3. Check Ignored/Direct Message Threads
List threads that may have been sent directly or were ignored:
```bash
node .agent/skills/read-email/scripts/read.js --query "to:help@opf.energy" --limit 5
```

## ⚠️ Troubleshooting Auth Errors

If the script fails, make sure your active local gcloud session has project access:
```bash
gcloud config set project opf-insight
```
